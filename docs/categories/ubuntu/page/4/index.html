Ubuntu 9.10 Beta

Ubuntu 9.10 Betaを試すため、virtualbox-ose 2.1.4を入れる。

そして以下からCDイメージを入れるが、動かない。

http://cdimage.ubuntu.com/daily/current/

Ubuntu側の問題かと思って以下からイメージをとってくるが

インストールまでしか進まず、起動しない。

http://releases.ubuntu.com/releases/9.10/

これはvirtualbox側の問題だろうとvirtualbox-oseをアンインストールして

以下から3.0.8をダウンロードして実行すると動いた。

http://www.virtualbox.org/wiki/Linux_Downloads

Opera

Operaをインストールするため

debファイルをダブルクリックしパッケージ・インストーラを起動しインストールしようとしたら

Synaptic パッケージマネージャと同時に起動しないでください。

というようなメッセージが出る。

そんなことをはしていないのに

そこでSynaptic パッケージマネージャを起動してみると

sudo dpkg –configure -aをしろというメッセージが出たので

行ったらパッケージ・インストーラを起動からインストールできるようになった。

Operaはデフォルトで日本語入力できなかったが

/usr/bin/operaにexport QT_IM_MODULE=XIMを追記することで日本語入力可能になった。

参考サイト

http://forums.fedoraforum.org/showthread.php?t=205725

画面解像度

VirtualBoxの画面解像度を変えないと作業がやりずらかったので

以下の手順で変更

1. Virtualboxのメニューから「デバイス」→「Guest Additions のインストール」

2. Virutualboxのメニューから「デバイス」→「CD/DVD-ROMのマウント」→「CD/DVD-ROMイメージ」→「VBoxGuestAdditions.iso」

3. ターミナル上で ‘/media/cdrom0/autorun.sh’

4. 3.が完了したら、再起動

参考サイト

http://blog.x28.jp/archives/35

Ubuntu 9.04アップグレード

Ubuntu 8.10から9.04へのアップグレード時の設定等のメモ

ネットワーク

ネットワークがつながらない。（回線はBフレッツです。）

ppoeconfしてみる→設定が残っている→そのまま進めるとネットワークにつながるようになった。

これは起動時につながらないようになっているだけ？

と思って

sudo gedit /etc/rc.localに以下を記述した所、自動的に接続されるようになった。 

ifconfig eth0 up
pon dsl-provider

opera

プラグインの設定を再設定する必要があった。

numlockの自動有効化を再設定

sudo gedit ‘/etc/gdm/Init/Default’

exit 0の上に以下を追記 

if [ -x /usr/bin/numlockx ]; then
/usr/bin/numlockx on
fi

sources.list

sudo gedit /etc/apt/sources.list

deb http://deb.opera.com/opera stable non-freeのコメントを外す

個人的な改善点

起動時に時々デスクトップのアイコンが表示されないことがあった。

それが改善された気がする。

起動してログアウトして再度ログインすると直っていたので大きな問題ではなかったのだが

改善されたようでうれしい。
