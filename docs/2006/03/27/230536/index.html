Haskellプログラミング入門（5）場合分け

Haskellのソースを読んでいても意味がわからない部分がまだ多い。

少しずつやっていこうということで、今日は場合分けを勉強した。

http://web.archive.org/web/20050216040914/www.teu.ac.jp/kougi/koshida/Prog6/text02.html

上記を参考にして2引数をとり最小の値を返す関数が以下になる。 

mymin :: Int -> Int -> Int
mymin x y
| x <= y      = x -- まずこれを評価条件に当てはまればx
| otherwise   = y -- 上記以外ならy
以下のようにもかける
mymin2 :: Int -> Int -> Int
mymin2 x y = if x <= y then x else y

3引数をとり最小の値を返す関数が以下になる。 

mymin3 :: Int -> Int -> Int -> Int
mymin3 x y z
| (x <= y) && (x <= z)    = x
| y <= z                  = y
| otherwise               = z

この場合分けは理解できればわかりやすいんだけど

始めてみた時は、これは分岐してるのか？

いったい何なんだ？という程度しか分からなかった。

少しずつソースが読めるようになってきた気がするので

頑張っていきたい。
