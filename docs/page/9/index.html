Opera

Operaをインストールするため

debファイルをダブルクリックしパッケージ・インストーラを起動しインストールしようとしたら

Synaptic パッケージマネージャと同時に起動しないでください。

というようなメッセージが出る。

そんなことをはしていないのに

そこでSynaptic パッケージマネージャを起動してみると

sudo dpkg –configure -aをしろというメッセージが出たので

行ったらパッケージ・インストーラを起動からインストールできるようになった。

Operaはデフォルトで日本語入力できなかったが

/usr/bin/operaにexport QT_IM_MODULE=XIMを追記することで日本語入力可能になった。

http://forums.fedoraforum.org/showthread.php?t=205725

Webアプリケーションのためのhtml5

html仕様書のAbstractを見ると

html4.01の主要な目的がWebの作成であったのに対し

html5の主要な目的がWebの作成＋Webアプリケーションのサポートとなっているように見える。

the publishing language of the World Wide Web

http://www.w3.org/TR/html401/

new features are introduced to help Web application authors

http://www.w3.org/TR/html5/

html4.01ではWebアプリケーションのことがあまり考慮されていなく

JavaScript等で強引に対応をしていた部分が多々ある。

それに対処するための改良案がhtml5ということ。

Webアプリケーションサポート以外の部分も含まれているが

Webアプリケーションサポートの部分はどうしても欲しい。

早く普及してくれ。

html5によってWebアプリケーションの底上げと更なる発展が期待できる。

今まで苦労していた部分がhtml5に取り込まれることによって

少ない労力でユーザフレンドリーなWebアプリケーションが用意できる。

さらにhtml5で行われていることの実現に傾けられていた労力が

更なる改善に向かうことになる。

HTML5

Webアプリケーションを作る際、入力値のバリデーション等が強化されて何がうれしいかというと

・JavaScriptの共通チェックがいらなくなる

　Webアプリケーションを作る際、共通的な入力値チェックについては

　そのアプリケーション共通で使えるようにJavaScriptを組むことがあると思うのですが、

　これがいらなくなるということです。

　これによりアプリケーション共通での作業が変わってくるでしょう。

　またバリデーションJavaScriptのバグによるチェックの漏れに悩まされなくなるというのもうれしい

　他のJavaScriptと干渉してチェックが行われないということがあり

　さらにそれを作ったチームはもうありませんのでできません直せませんといわれると厳しい。

・JavaScriptの個別バリデーションを減らすことが出来る。

　いままで個別にバリデーションJavaScriptを組む必要があったものが不要になる可能性が増える。

　正規表現を引数にとるバリデーション用JavaScriptを組めば、今でもある程度負荷は軽減できるけど

　number等を使えば数字しか入力できないというようにhtmlのみでバリデーションできるのがよい。

　これにより負荷の軽減ができる。pattern属性、min属性、max属性等もうれしい。

・画面の表示が早くなる

　JavaScriptでチェックを行うとJavaScriptのファイル分読み込みが遅くなるため

　それがなくなると早くなる。

・操作がわかりやすくなる

　今まで日付入力の際のカレンダーを独自に作る事があったと思うのですが、

　それがブラウザで共通化されたものを使えるようになり、

　オペレーションが共通化されるため、操作を類推しやすくなる。
